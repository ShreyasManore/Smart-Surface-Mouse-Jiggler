# Working Principle

## Overview

The Smart Surface Mouse Jiggler prevents computer sleep by generating physical surface motion beneath an optical mouse, causing the mouse's own sensor to report continuous movement to the host operating system — exactly as if a person were gently moving it.

## Detailed Operation

### 1. Power Stage

Closing the ON/OFF switch completes the circuit between the 18650 battery and the DC gear motor. Current flows directly from the battery through the switch to the motor with no intermediate regulation in the baseline design.

### 2. Mechanical Actuation

The motor's geared output shaft rotates at a low, steady RPM (typically 10–30 RPM). This shaft is directly coupled to a textured disc mounted flush with the top of the enclosure.

### 3. Optical Detection

An optical mouse sensor works by capturing thousands of small images of the surface beneath it per second and comparing successive frames to calculate directional displacement — the same principle used to track a mouse across a desk. When the mouse rests on the slowly rotating textured disc instead of a stationary desk, the disc's rotation is optically indistinguishable from very slow, continuous manual movement of the mouse itself.

### 4. Host System Response

The mouse driver reports this detected motion as standard HID (Human Interface Device) movement events — no different from normal use. The operating system's idle timer, which tracks user input activity, is reset continuously, preventing sleep, screen lock, or screensaver activation.

## Why This Works Without Software

Operating systems determine "user activity" primarily through input events — keyboard presses and mouse movement/clicks — received through the standard HID stack. Because this device produces genuine HID-level mouse movement events (generated by the mouse's own unmodified hardware), it requires no OS-level permissions, no driver installation, and cannot be detected or blocked the way a software-based jiggler or emulated USB HID device might be by endpoint security software.

## Key Physical Relationship

```
Motor RPM  →  Disc Surface Velocity  →  Detectable Frame-to-Frame Delta  →  Registered Mouse Movement
```

If disc rotation speed is too low, frame-to-frame displacement may fall below the sensor's detection threshold. If too high, it may register as unrealistically fast movement. The tuned operating range (10–30 RPM on a ~70mm disc) was empirically validated in [`docs/Testing_Report.md`](../docs/Testing_Report.md) to reliably register as continuous, natural-looking motion.
